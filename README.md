## Bias Evaluation Pipeline

This repo runs text generation using a chosen LLM on a set of bias-test prompts, evaluates generations for toxicity + sentiment, aggregates results into summary tables, and visualizes them in a Streamlit dashboard.

It supports multiple prompt sources:
- Local JSONL (`--prompt-source jsonl`)
- AllenAI Real Toxicity Prompts / RTP (`--prompt-source rtp`)
- CrowS-Pairs (`--prompt-source crows`)

## High-level Architecture

```mermaid
flowchart TD
  U[User] -->|CLI| INF[run_inference.py]
  U -->|Browser| DASH[Streamlit Dashboard]

  INF -->|loads prompts| PS[Prompt Sources]
  PS --> RTP[RTP HF dataset]
  PS --> CROWS[CrowS-Pairs loader]
  PS --> JSONL[Local prompts.jsonl]

  INF --> GEN[LLM generation<br/>Transformers]
  GEN --> MET[metrics.py<br/>toxicity + sentiment]
  MET --> OUTRUNS[outputs/runs/<run_id>.jsonl]

  OUTRUNS --> AGG[aggregate.py<br/>group stats + gaps + enriched jsonl]
  AGG --> OUTSUM[outputs/summaries/<run_id>.summary.csv]
  AGG --> OUTGAPS[outputs/summaries/<run_id>.gaps.csv]
  AGG --> OUTENR[outputs/summaries/<run_id>.enriched.jsonl]

  AGG --> PLOT[report.py<br/>optional charts (png)]
  PLOT --> PLOTDIR[outputs/plots/]

  DASH -->|reads| OUTSUM
  DASH -->|reads| OUTENR

  DASH -->|button click| GEM[Gemini API call<br/>summarize enriched.jsonl]
  GEM -->|render markdown| DASH
```

## Outputs

After a run + aggregation:
- `outputs/runs/<run_id>.jsonl`: raw generations + prompt metadata
- `outputs/summaries/<run_id>.summary.csv`: aggregated metrics by `model/decoding/category/attribute`
- `outputs/summaries/<run_id>.gaps.csv`: bias gap metrics (max-min across attributes within a category)
- `outputs/summaries/<run_id>.enriched.jsonl`: enriched per-generation rows including computed `toxicity` and `sentiment`
- `outputs/plots/*.png`: optional plots generated by `src/report.py`

## Setup

Install dependencies:

```bash
pip install -r requirements.txt
```

If you want the dashboard “AI summary (Gemini)” feature, set:
- `GEMINI_API_KEY` (see `.env`)

## Prompt Sources

### JSONL
Use `--prompt-source jsonl` with:
- `--prompts data/prompts/prompts.jsonl`

The file is expected to be JSONL where each row looks like:
- `{"id": "...", "category": "...", "variants": [{"attribute": "...", "prompt": "..."}]}`

### RTP (Real Toxicity Prompts)
Use `--prompt-source rtp` and set:
- `--rtp-max-prompts`
- `--rtp-seed`

### CrowS-Pairs
Use `--prompt-source crows` and set:
- `--crows-split` (default: `test`)
- `--crows-max-prompts`
- `--crows-seed`

## Running Inference

From repo root, run:

```bash
python3 -m src.run_inference \
  --prompt-source crows \
  --crows-max-prompts 50 \
  --crows-seed 42 \
  --crows-split test \
  --model distilgpt2 \
  --samples 5 \
  --out_dir outputs/runs
```

Other examples:

RTP:
```bash
python3 -m src.run_inference \
  --prompt-source rtp \
  --rtp-max-prompts 30 \
  --rtp-seed 42 \
  --model distilgpt2 \
  --samples 5 \
  --out_dir outputs/runs
```

Local JSONL:
```bash
python3 -m src.run_inference \
  --prompt-source jsonl \
  --prompts data/prompts/prompts.jsonl \
  --model distilgpt2 \
  --samples 5 \
  --out_dir outputs/runs
```

## Aggregating + Summarizing

Once inference has produced a run JSONL:

```bash
python3 -m src.aggregate \
  --run outputs/runs/<run_id>.jsonl \
  --out_dir outputs/summaries \
  --device -1
```

## Dashboard (Streamlit)

Run:

```bash
streamlit run app/streamlit_app.py
```

The dashboard:
- loads `outputs/summaries/*.summary.csv`
- renders summary tables and charts
- lets you explore per-generation rows from `outputs/summaries/<run_id>.enriched.jsonl`

## Gemini “AI summary” Button

In the dashboard’s enriched section, there is a button that:
1. reads the selected run’s `outputs/summaries/<run_id>.enriched.jsonl`
2. sends its contents to Gemini
3. renders Gemini’s returned bullet summary

Set `GEMINI_API_KEY` via environment or `.env`.

## Evaluation Models

The per-generation evaluation (toxicity + sentiment) is computed in `src/metrics.py` by the `Evaluators` class.

- Toxicity model: `unitary/toxic-bert` (Hugging Face `text-classification`)
- Sentiment model: `distilbert-base-uncased-finetuned-sst-2-english` (Hugging Face `text-classification`)

Label-to-score mapping is handled in `src/metrics.py`:
- Toxicity: attempts to interpret the classifier label as toxic vs non-toxic, then converts to a “prob toxic” score
- Sentiment: maps `POSITIVE` to high score and everything else to `1 - score`

To change these models, edit the default arguments in `Evaluators.__init__` inside `src/metrics.py`.

## Notes

- Some Hugging Face datasets rely on legacy dataset scripts; if loading a prompt source fails, the CrowS-Pairs loader includes a fallback to the canonical CSV.
- `run_inference` uses `transformers` model generation and the repo’s `metrics.py` for scoring.
